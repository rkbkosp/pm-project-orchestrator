# Meeting Flow

Use this when the user starts with a rough idea and not a mature document.

## Meeting goal
Turn a vague idea into a product blueprint that is specific enough for prototype design and later implementation planning.

## Agenda

### 1. Objective clarification
Capture:
- what the user wants to build
- why they want to build it
- what success looks like

### 2. Target users and scenarios
Capture:
- who the product is for
- when they would use it
- what current pain or workaround exists

### 3. Value and differentiation
Capture:
- what the core value is
- why someone would adopt it
- what makes it worth building now

### 4. MVP scope and non-goals
Capture:
- what the first version must do
- what will not be included yet
- what would make the first version too broad

### 5. Unknowns and risks
Capture:
- biggest uncertainty
- biggest product risk
- biggest implementation or adoption risk

### 6. Conclusion
Output:
- one-sentence product definition
- top user scenario
- mvp summary
- non-goals
- open questions
- recommendation for next stage

## Moderation guidance
- Run this like a structured project kickoff, not a casual brainstorm.
- Ask only the minimum follow-up needed to unblock the blueprint.
- Periodically summarize decisions so the user can correct them.
- If the user is nontechnical, do not push technical architecture questions into this meeting unless they materially affect product scope.
