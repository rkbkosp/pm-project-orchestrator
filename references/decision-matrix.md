# Decision Matrix

Use this template whenever a technical choice matters at the product level.

| Option | What this means for the PM | Advantages | Tradeoffs / cost | Best fit | Recommendation level |
|---|---|---|---|---|---|

## Required follow-up summary

### Recommended option
State one clear recommendation.

### Why
Explain in nontechnical language. Focus on timeline, user experience, maintainability, and future iteration.

### Safe default if the user is unsure
Offer the most conservative reasonable choice.

## Example prompts that should trigger this
- choosing between faster delivery and stronger long-term architecture
- deciding whether to build one generalized module or separate modules
- deciding whether a workflow should be local-only, cloud-assisted, or fully cloud-based
- deciding whether a first release should include automation or manual fallback
